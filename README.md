# Type Battle



## 必要な環境

- Go 1.25以降
- LinuxまたはWSLでウィンドウを表示できる環境
  - Windows 11のWSLではWSLgを利用できます

## Linux / WSLのセットアップ

Ebitengineのビルドに必要なX11、OpenGL、音声関連の開発パッケージをインストールします。

```bash
sudo apt update
sudo apt install -y gcc libc6-dev libgl1-mesa-dev libx11-dev libxcursor-dev libxi-dev libxinerama-dev libxrandr-dev libxxf86vm-dev libasound2-dev pkg-config
```

コマンドを複数行に分ける場合、行末のバックスラッシュ (`\`) の後ろに空白を入れないでください。貼り付け時のエラーを避けるには、上記の一行コマンドをそのまま使用してください。

## 実行

プロジェクトのディレクトリで次を実行します。

```bash
go run .
```

アプリケーションは画面サイズに合わせてフルスクリーン表示されます。左側の200pxを管理メニューとして使用し、残りの表示領域いっぱいにグリッドを伸縮して表示します。起動時に18種類のタイプを同数ずつ生成してランダムに配置します。セル総数を18で割った余りは、ランダムに選んだタイプへ割り振られます。管理メニューには現在の`TIME`、グリッドの行・列数、タイプごとの色見本とセル数が表示されます。

`TIME`は1から始まり、Spaceキーを押すたびに1つ進みます。

左メニューの`AUTO PLAY`ボタンをクリックすると自動再生のON/OFFを切り替えられます。ONの間は1秒間に10 TIME進みます。

TIMEが進むと、各セルは上下左右にあるセルから攻撃対象をランダムに1つ選びます。盤面の外側は攻撃対象に含まれません。18タイプの相性で「こうかはばつぐん」となる場合、攻撃されたセルが攻撃者と同じタイプへ変化します。同じTIMEの攻撃は更新前の盤面を基準に判定し、結果を一括で反映します。

1つのセルに異なるタイプから複数の有効な攻撃が届いた場合は、成功した攻撃の中からランダムに1つを選びます。攻撃1回を1票として扱うため、地面から1回、水から3回攻撃された場合は、地面になる確率が1/4、水になる確率が3/4です。

グリッドの列数と行数は、`main.go`の`gridColumns`と`gridRows`で変更できます。

## トラブルシューティング

### `X11/Xlib.h: No such file or directory`

Linux向けの開発パッケージが不足しています。「Linux / WSLのセットアップ」に記載した`apt install`を実行してください。

### ウィンドウが表示されない

WSLから実行している場合は、WSLgなどのGUI表示環境が利用できることを確認してください。
